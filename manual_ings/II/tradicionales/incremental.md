# Incremental

El sistema se divide en componentes funcionales. A continuación, se desarrollan esos componentes uno por uno con el objetivo de perfeccionar cada componente antes de comenzar a implementar el siguiente.

Por ejemplo, un equipo que desarrolla una plataforma de gestión mediante una metodología incremental debe comenzar identificando las características que componen el núcleo de la aplicación. A continuación, establecerá otros componentes: acceso para nuevos tipos de usuario, generación de informes y estadísticas.... Una vez dividido el proyecto en componentes funcionales, se irán implementando uno por uno.

## Ventajas
- Se obtiene software funcional (aunque limitado) desde la primera entrega.
- Menos lineal y menos riesgo que en cascada.
- Detección temprana de problemas.

## Desventajas
- Planificación más compleja.
- Puede haber problemas de integración entre incrementos.
- No siempre se identifican bien los requisitos de un componente desde el inicio.
